                  Emits K sequential FORCE_DELTA copies joined by CONCAT+CONSOLIDATE

                  Emits single WL_PLAN_OP_K_FUSION operator with K internal sequences;

NOTE: K-FUSION mode currently executes K sequences sequentially (not in parallel).
Parallel workqueue dispatch is scaffolded (col_op_k_fusion_worker) but not yet
wired to the workqueue. Performance parity with sequential is therefore expected.

ANALYSIS & NEXT STEPS
---------------------
The K-FUSION operator infrastructure is complete and correct:
  - Plan generation: WL_PLAN_OP_K_FUSION nodes emitted correctly
  - Backend execution: K sequences evaluated, results merged with deduplication
  - Correctness: identical tuple counts and iteration counts vs sequential

Current limitation: K sequences are evaluated sequentially, not in parallel.
The worker context (col_op_k_fusion_worker_t) and worker function
(col_op_k_fusion_worker) are implemented but the workqueue dispatch loop
is not yet wired in col_op_k_fusion().

Expected speedup when parallelism is enabled: up to K-fold for compute-bound rules
on machines with K+ cores. For K=2 (memoryAlias rule), expected ~1.5-1.8x speedup
on a 2-core system (memory bandwidth is often the bottleneck).
